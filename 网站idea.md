# 个人网页方案总结：鹅鸭杀主题 · 点击移动地图版

---

## 一、项目定位

一个带有 **“爱看鹅鸭杀直播的小姐姐一枚”** 标签的个人网站，用游戏化的两层结构来展示你的经历、兴趣与个性。整体风格轻松、幽默，保留“vibe coding”的随性与趣味，但不追求复杂的游戏机制，重点放在内容展示和轻交互上。

---

## 二、整体结构

### 第一层：身份牌入口
- **视觉**：一张仿鹅鸭杀角色卡牌，正面是你选定的游戏角色插画/图标。
- **文案**：用游戏风格介绍你的“角色属性”，实际是个人关键词的提炼。
- **按钮**：下方有一个“进入游戏”按钮，点击后切换到第二层地图界面。
- **可选元素**：背景氛围、加载动画（模拟游戏加载）。

### 第二层：个人经历地图（点击移动版）
- **视觉**：俯视角地图，划分为若干区域，每个区域代表你人生的一个方面。
- **角色**：地图上有一个可移动的小角色（如一个小人偶或鹅的图标）。
- **交互**：用户点击某个区域，角色自动移动到该区域（带有简单动画），到达后弹出信息窗，展示该板块对应的个人经历、技能、故事等。
- **无需键盘控制**：全程鼠标点击，手机端同样友好。

---

## 三、身份牌设计

### 3.1 角色选择
从鹅鸭杀角色中选一个最能代表你性格或经历的角色，也可以自创一个。建议方向：

| 角色 | 映射含义 |
|------|----------|
| **网红鸭** | 擅长组织、带动气氛，有影响力 |
| **侦探** | 观察敏锐，擅长发现问题 |
| **呆呆鸟** | 表面迷糊，实则关键时刻反转 |
| **模仿鹅** | 适应力强，能切换多重身份 |
| **自定义** | 如“观战小姐姐”——弹幕预言家，永远在吃瓜 |

## 四、地图设计

### 4.1 地图区域划分（推荐 4 个，可自定义）

| 区域名称 | 代表内容 | 点击后展示的信息 |
|----------|----------|------------------|
| **🌲 代码森林** | 技术/工作经历 | 职业、技能栈、项目经历、技术趣事 |
| **☕ 深夜咖啡馆** | 兴趣/日常/随想 | 其他爱好、MBTI、书影音、生活片段 |
| **🏆 成就殿堂** | 高光时刻 | 引以为傲的事（可大可小，如完成某个项目、连续30天早起） |

> 你也可以根据实际情况增减区域或改名，比如增加“学习小屋”“旅行地图”等。

### 4.2 地图视觉风格
- **背景**：手绘风格的地图底图，或使用简单的网格/草地纹理。
- **区域标识**：每个区域有明显的图标和名称（如一个帐篷+文字）。
- **角色**：一个可爱的像素风小人或鹅鸭杀风格的小动物，放置在地图的某个起始点。
- **交互反馈**：鼠标移到区域上时高亮；点击后角色移动并弹窗。

### 4.3 弹窗内容格式（轻量化）
弹窗采用简洁的卡片形式，内容为纯文本 + 少量列表，不涉及复杂表单。示例：

```
🌲 代码森林

职业：前端开发 / 3年经验
技能：React · Tailwind · 与bug斗智斗勇

做过的事：
- 参与xx项目重构，性能提升30%
- 开发鹅鸭杀弹幕助手（个人小玩具）

一句话：“写代码就像在鹅鸭杀里当好人——永远在猜bug藏在哪里。”
```

## 五、交互流程（用户视角）

1. 打开网页，看到身份牌，阅读角色介绍。
2. 点击“进入游戏”，页面切换（或淡入淡出）到地图界面。
3. 地图中央/左下角有一个小角色，周围分布着各个区域。
4. 用户点击“代码森林”区域 → 角色平滑移动到该区域上方 → 弹出信息窗显示你的技术经历。
5. 关闭弹窗后，用户可以继续点击其他区域，角色再次移动并展示相应内容。
6. 所有区域的内容均代表你的真实经历与个性。

## 五、地图界面样式

### 5.1 整体布局
- **顶部状态栏**：仿游戏 UI，显示“当前对局：人生Online | 角色：模仿鹅 | 状态：探索中”。
- **地图区域**：使用 **CSS Grid** 布局，3 个主要区域 + 可能的路径装饰。每个区域为一个卡片，可点击。
- **角色**：绝对定位的圆形或小动物图标，尺寸 `48x48`，带轻微阴影。

### 5.2 地图区域卡片设计
- **尺寸**：`200x200`（桌面端），`160x160`（平板），`自适应`（手机）。
- **背景**：半透深色 + 毛玻璃（`backdrop-filter: blur(4px)`）。
- **边框**：`1px solid rgba(59,130,246,0.3)`，悬浮变为 `#3B82F6`。
- **图标**：每个区域一个 emoji 或简单 SVG（🌲、☕、🏆）。
- **标题**：居中，字体粗，带轻微发光效果。
- **悬浮效果**：卡片放大 `scale(1.02)`，边框变色，阴影加深。

### 5.3 地图背景与装饰
- **背景**：深蓝灰底色 + 网格线（`background-image: linear-gradient` 绘制），模拟游戏地图的格子。
- **装饰元素**：随机放置几个小物件（爆米花桶、呆鸭、咖啡杯）作为彩蛋，绝对定位，点击时 alert 一句俏皮话。

---

## 六、角色移动动画

- **移动方式**：点击区域卡片后，角色使用 `transition` 平滑移动到卡片中心。
- **CSS 实现**：角色 `position: absolute`，动态更新 `top`/`left`，加 `transition: all 0.3s ease-out`。
- **到达效果**：角色轻微弹跳（`scale` 动画），然后弹出信息窗。
- **移动时禁止重复点击**：移动过程中禁用点击，到达后恢复。

---

## 七、弹窗样式

- **触发**：角色到达区域后自动弹出。
- **样式**：
  - 固定大小 `min-width: 280px`，`max-width: 500px`，背景 `#1E293B`，圆角 `20px`，边框 `1px solid #3B82F6`。
  - 标题栏：区域图标 + 区域名称 + 关闭按钮（X）。
  - 内容区域：`max-height: 60vh`，滚动条自定义。
  - 关闭方式：点击关闭按钮、点击遮罩层（如果有）、按 ESC。
- **动画**：淡入 + 上浮（`fadeInUp`），时长 0.2s。

## 八、氛围增强元素

### 8.1 状态栏
- 位置：页面顶部固定，半透背景，圆角底部。
- 内容：动态显示当前区域（可选），或固定显示游戏状态。
- 样式：`font-family: 'Press Start 2P', monospace`，小字号，霓虹蓝文字。

### 8.2 背景音乐开关
- 位置：右下角固定按钮。
- 样式：圆形，音符图标，悬浮变色。
- 功能：点击播放/暂停 Lo-fi 或游戏大厅风格 BGM（使用 `<audio>` 元素）。

### 8.3 小彩蛋
- 地图上放置 2-3 个可点击装饰物（如爆米花桶、呆鸭、咖啡杯）。
- 点击后显示短文本（例如：“这桶爆米花已经陪你看完 100 场直播了”）。
- 实现简单：绝对定位，`cursor: pointer`，`alert` 或自定义小弹窗。

## 九、响应式与性能

- **桌面端**：地图区域采用 Grid，3 列布局，角色移动坐标基于相对位置。
- **平板/手机**：
  - 地图改为垂直排列（或 2 行 2 列），角色移动可能需要重新计算坐标（使用百分比）。
  - 弹窗占屏幕 90% 宽度，边距适中。
  - 状态栏字体缩小。
- **性能**：CSS 动画使用 `transform` 和 `opacity`，避免 `top/left` 动画触发重排（角色移动可用 `transform: translate`）。

## 十、实现优先级（按开发顺序）

1. **身份牌页面**：HTML/CSS 布局 + 进入游戏按钮。
2. **地图页面基础结构**：3 个区域卡片 + 角色占位 + 状态栏。
3. **点击移动逻辑**：点击区域 → 计算目标位置 → 移动角色 → 触发弹窗。
4. **弹窗内容填充**：使用已定文案。
5. **氛围增强**：背景音乐、彩蛋、音效（可选）。
6. **响应式调整**。
